# 06 最佳实践

## 推荐做法

1. 先定规格，再让 AI 写代码。
2. 让测试成为实现前置条件，而不是事后补丁。
3. 把 AI 的行为规则和项目事实分开维护。
4. 尽量让每次变更都留下可归档文档。
5. 对已有 AI 配置默认采用保守安装策略。

## 常见反模式

1. 直接让 AI 大改代码，但没有 proposal 或 design。
2. 没有测试或验收标准，就让 AI 判断“完成”。
3. 把项目事实写进 `AGENTS.md`，导致规则和配置纠缠。
4. 让安装流程默认覆盖已有 AI 目录。
5. 一边改 schema，一边不校验阶段定义。

## 维护建议

- 修改 `stages/*.yaml` 后运行校验脚本
- 变更命令语义时同步更新 README 和 guide
- 把“规则”“配置”“流程”“执行入口”分层维护
