# 错误恢复指南

> 本文档提供安装过程中的错误处理和回退策略。

---

## 错误类型分类

| 类型 | 严重程度 | 是否可恢复 | 处理方式 |
|------|----------|------------|----------|
| **前置条件失败** | 高 | 是 | 引导用户安装缺失项 |
| **目录操作失败** | 高 | 是 | 检查权限/路径，重试 |
| **模板复制失败** | 高 | 是 | 检查模板路径，恢复原文件 |
| **变量填充失败** | 中 | 是 | 询问用户手动填充 |
| **配置覆盖失败** | 高 | 部分 | 从备份恢复 |
| **网络错误** | 中 | 是 | 提供离线方案 |

## 前置条件错误

### 错误：OpenSpec CLI 未安装

**检测时机**：前置要求检查阶段

**错误信息**：
```
❌ 未检测到 OpenSpec CLI
```

**恢复步骤**：

```
错误恢复方案：

【选项 A】自动全局安装
   执行：npm install -g @fission-ai/openspec@latest
   安装后运行：openspec --version

【选项 B】手动安装
   1. 访问：https://github.com/Fission-AI/OpenSpec/
   2. 按照官方指南安装
   3. 安装完成后重新运行本指南

【选项 C】取消安装
   请稍后手动完成安装后再运行

请选择（A/B/C）：
```

**命令**：
```bash
# 选项 A 执行
npm install -g @fission-ai/openspec@latest

# 验证
openspec --version
```

### 错误：Node.js 版本不兼容

**检测时机**：前置要求检查阶段

**错误信息**：
```
❌ Node.js 版本过低：当前 v16.x，要求 >= 18.x
```

**恢复步骤**：

```
错误恢复方案：

OpenSpec CLI 需要 Node.js 18.x 或更高版本。

当前版本：v16.x
要求版本：>= 18.x

【选项 A】使用 nvm 安装
   执行：nvm install 22 && nvm use 22

【选项 B】使用 n 更新
   执行：n 22

【选项 C】手动安装
   1. 访问：https://nodejs.org/
   2. 下载并安装 LTS 版本
   3. 重启终端后重新运行本指南

请选择（A/B/C）：
```

### 错误：文件覆盖失败

**检测时机**：阶段 5.3 文件复制

**错误信息**：
```
❌ 无法覆盖文件：<文件路径>
   文件被占用或权限不足
```

**恢复步骤**：

```
错误恢复方案：

文件覆盖失败。

【选项 A】备份后重试
   将备份原文件到 .backup/<时间戳>/
   然后重新复制

【选项 B】跳过该文件
   继续复制其他文件
   注意：可能导致配置不完整

【选项 C】取消安装
   稍后手动解决冲突后重试

请选择（A/B/C）：
```

**备份命令**：
```bash
# 创建备份
mkdir -p .backup/$(date +%Y%m%d_%H%M%S)
cp <文件路径> .backup/$(date +%Y%m%d_%H%M%S)/

# 重试复制
cp <源文件> <目标路径>
```

## 安装回滚

### 完全回滚

如果安装过程中出现严重错误，需要完全回滚：

```bash
# 1. 备份现有文件（如有）
if [ -d .backup ]; then
  # 从备份恢复
  cp -r .backup/* ./
fi

# 2. 删除 OpenSpec 创建的文件
rm -rf openspec/
rm -f AGENTS.md
rm -rf .opencode/commands/
rm -rf .opencode/skills/

# 3. 删除 AI 配置目录（如果是新创建的）
# 注意：仅在确定是新创建时执行
# rm -rf .opencode/

# 4. 恢复 Git 状态（如有未提交的更改）
git checkout -- .

# 5. 清理备份
rm -rf .backup/
```

### 部分回滚

如果只是某一阶段失败：

```bash
# 示例：回滚阶段 5.3 的文件覆盖
# 从 .backup 恢复特定文件
cp .backup/*/AGENTS.md ./
cp .backup/*/openspec/config.yaml ./openspec/
```

## 错误日志

建议在安装过程中生成错误日志：

```bash
# 创建日志目录
mkdir -p openspec/logs

# 记录错误
echo "[$(date)] ERROR: <错误信息>" >> openspec/logs/install.log

# 记录成功步骤
echo "[$(date)] SUCCESS: <步骤描述>" >> openspec/logs/install.log
```

## 常见问题

### Q: 安装中断后如何继续？

```
A: 检查 openspec/logs/install.log 了解中断位置
   根据中断阶段，从对应阶段重新开始
   或选择完全回滚后重新安装
```

### Q: 如何恢复被覆盖的自定义配置？

```
A: 如果选择了备份，检查 .backup/ 目录
   从备份中恢复需要的文件
   重新整合到新配置中
```

### Q: CI/CD 配置失败会影响核心功能吗？

```
A: 不会。核心功能不依赖 CI/CD
   可以稍后手动配置
   或使用 Manual 模式手动执行
```
