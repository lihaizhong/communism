# 强制停止点检查清单

> 本文档提供各停止点的快速检查清单，AI 在每个停止点必须暂停并等待用户确认。

---

## 🛑 停止点概览

| 停止点 | 位置 | 必须确认的内容 |
|--------|------|----------------|
| **停止点 1** | 阶段 4.1 | AI 配置目录选择 |
| **停止点 2** | 阶段 4.4 | 已有 AI 文档整合 |
| **停止点 3** | 阶段 5.3 | 文件覆盖确认 |

---

## 停止点 1：AI 配置目录选择

### 触发条件

执行完阶段 4.1 扫描配置目录后。

### 检查内容

```
□ 已完成配置目录扫描
□ 列出所有检测到的 AI 配置目录
□ 展示覆盖警告
□ 等待用户选择目标目录
```

### 必须向用户展示

```
📁 AI 配置目录检测

检测结果：
├─ 检测到 [0/1/2+] 个已有 AI 配置目录
│   ├─ .opencode/    (OpenCode)
│   ├─ .claude/      (Claude Code)
│   ├─ .cursor/      (Cursor)
│   └─ .github/copilot/ (Copilot)
│
└─ 请选择目标目录：[列表/新建目录]

⚠️ 覆盖警告：
如果选择更新已有目录，以下内容将被完全替换：
• commands/ 目录（所有 .md 命令文件）
• skills/ 目录（所有技能目录）
• 已有自定义配置将丢失

建议：如有自定义配置，请先备份或选择创建新目录。

请选择：
[ ] 使用检测到的目录：___________
[ ] 创建新目录：___________
[ ] 取消安装
```

### 禁止行为

- ❌ 扫描后直接继续到下一步
- ❌ 假设用户选择而不等待确认
- ❌ 在用户未选择前执行任何覆盖操作

### 用户确认后

记录用户选择的目标目录，继续执行步骤 4.2（检测当前运行的 AI）。

---

## 停止点 2：已有 AI 文档整合

### 触发条件

阶段 4.4 检测到已有 AI 文档后。

### 检查内容

```
□ 已完成 AI 文档检测
□ 列出所有检测到的文件
□ 展示信息提取预览
□ 等待用户确认是否整合
```

### 必须向用户展示

```
📄 已有 AI 文档检测

检测到的文件：
├─ AGENTS.md          (OpenCode / 通用)
├─ CLAUDE.md          (Claude Code)
├─ IFLOW.md           (iFlow)
├─ CURSOR_RULES.md    (Cursor)
├─ .cursorrules       (Cursor)
└─ .github/copilot-instructions.md (Copilot)

从上述文件中可提取：
├─ 技术栈信息 → 用于填充配置变量
├─ 编码规范 → 可整合到 AGENTS.md
├─ AI 角色定义 → 可整合到 AGENTS.md
└─ 项目约定 → 可整合到 AGENTS.md

⚠️ 注意：
• 技术栈信息将与阶段 3 检测结果对比，不一致时以检测结果为准
• 非技术类信息（规范、角色、约定）可选择性整合
• 如不整合，原有内容将被模板覆盖

请选择：
[ ] 整合非技术类信息到新配置
[ ] 不整合，使用 OpenSpec 默认模板
[ ] 取消安装
```

### 禁止行为

- ❌ 检测后直接继续
- ❌ 自动假设用户选择

### 用户确认后

- 选择「整合」→ 暂存非技术类信息，继续阶段 5
- 选择「不整合」→ 丢弃内容，继续阶段 5

---

## 停止点 3：文件覆盖确认

### 触发条件

阶段 5.3 执行文件覆盖前。

### 检查内容

```
□ 用户已在步骤 4 确认目标目录
□ 显示即将覆盖的文件列表
□ 展示不可恢复警告
□ 等待用户明确确认
```

### 必须向用户展示

```
⚠️ 文件覆盖确认

目标目录：[用户在步骤 4 选择的目录]

即将执行以下覆盖操作：
├─ openspec/config.yaml        （覆盖）
├─ openspec/schemas/           （覆盖）
├─ AGENTS.md                   （覆盖，如存在）
├─ commands/ 目录              （覆盖）
│   ├─ opsx-explore.md
│   ├─ opsx-propose.md
│   ├─ opsx-apply.md
│   ├─ opsx-archive.md
│   ├─ opsx-bugfix.md
│   └─ opsx-spike.md
└─ skills/ 目录                （覆盖）
    ├─ openspec-explore/
    ├─ openspec-propose/
    ├─ openspec-apply-change/
    ├─ openspec-archive-change/
    ├─ openspec-bugfix/
    └─ openspec-spike/

⚠️ 已有配置将被完全替换，无法恢复。

请确认：
[ ] 确认继续
[ ] 取消安装
[ ] 备份后继续（将创建备份到 .backup/ 目录）
```

### 禁止行为

- ❌ 未确认直接复制文件
- ❌ 假设用户已经同意
- ❌ 在用户未明确选择前执行任何覆盖

### 用户确认后

- 选择「确认继续」→ 执行步骤 5.3.1（复制模板文件）
- 选择「备份后继续」→ 创建备份，然后执行步骤 5.3.1
- 选择「取消」→ 终止安装流程

---

## 验证规则

每个停止点必须满足以下条件才能继续：

1. ✅ 已向用户展示完整信息
2. ✅ 已说明选择后果（覆盖警告）
3. ✅ 已收到用户明确选择（按钮、输入等）
4. ✅ 已记录用户选择

如果用户未选择：
- 必须**保持当前位置等待**
- 不能假设默认值
- 不能自动继续
